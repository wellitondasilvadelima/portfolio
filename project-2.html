PROJETO: LEITOR DE INVOICE PDF

Este projeto consiste em um software automatizado para leitura de invoices em PDF, extração de informações relevantes e armazenamento dessas informações em um banco de dados MySQL. Ele tem como objetivo agilizar e otimizar o processamento de documentos fiscais, eliminando a necessidade de entrada manual de dados e reduzindo possíveis erros humanos.

DESCRIÇÃO DO PROJETO

Muitas empresas lidam diariamente com uma grande quantidade de notas fiscais e invoices. O processo de entrada manual dessas informações em um banco de dados pode ser demorado, sujeito a erros e ineficiente. O projeto visa solucionar esses problemas por meio da automação, garantindo que os dados extraídos sejam precisos e armazenados corretamente para futuras consultas e análises.

O software lê os arquivos PDF da pasta input, extrai os dados relevantes (como número da invoice, data, valor, itens e pagamento), e os salva no banco de dados. O sistema também faz a organização dos arquivos, movendo os arquivos processados corretamente para a pasta data, e os arquivos que apresentaram erro para dataerror. Além disso, um arquivo Excel é gerado na pasta output com o log do processamento.

TECNOLOGIAS UTILIZADAS NO PROJETO

PyODBC (Conexão com o banco de dados): O PyODBC é utilizado para estabelecer a comunicação entre o Python e o banco de dados MySQL via driver ODBC. Ele permite executar queries SQL, inserir, atualizar e recuperar informações das invoices.

MySQL (Banco de Dados): O sistema utiliza MySQL para armazenar os dados extraídos das invoices. O banco de dados database_invoices é criado automaticamente caso não exista, e contém as seguintes tabelas:
- Invoices: Armazena informações gerais das invoices (número, data, valor total, etc.).
- Invoices_Items: Contém os detalhes dos itens de cada invoice.
- Payments: Registra informações de pagamento associadas às invoices.

Pandas (Processamento e Log): A biblioteca Pandas é usada para manipulação e tratamento de dados. Além disso, ela permite gerar um arquivo Excel que registra os logs do processamento, contendo informações como: Nome do documento, Data e hora do processamento, Nota extraída, Status do processamento (sucesso ou erro)

CustomTkinter (Interface Gráfica): O CustomTkinter é uma biblioteca baseada no Tkinter, que permite a criação de uma interface moderna e responsiva. O usuário pode interagir com o software de maneira simples, apenas clicando em um botão para iniciar o processamento.

SOBRE O SOFTWARE

Interface Intuitiva: O usuário não precisa de conhecimento técnico para operar o sistema. Basta selecionar os arquivos ou deixar os documentos na pasta input e clicar em um botão para iniciar o processamento. A interface foi projetada para ser simples, eficiente e visualmente agradável.

Persistência dos Dados no Banco de Dados: Todas as informações extraídas são armazenadas no MySQL, garantindo que possam ser consultadas posteriormente. Se uma invoice já foi processada anteriormente, os dados são atualizados para evitar duplicidade.

Log da Automação: Cada execução do software gera um relatório detalhado em Excel na pasta output, contendo informações essenciais sobre o processamento. Isso permite que o usuário acompanhe quais arquivos foram processados com sucesso e quais apresentaram erros

Este projeto proporciona eficiência e precisão no tratamento de invoices em PDF, garantindo que os dados sejam extraídos corretamente e armazenados de forma segura. Com uma interface intuitiva e um banco de dados estruturado, a automação desse processo economiza tempo e reduz erros manuais, tornando-se uma solução ideal para empresas que lidam com grande volume de documentos fiscais.
